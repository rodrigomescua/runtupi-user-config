## What This Repo Is

A Runtipi user-config repository for running self-hosted apps with a direct internet connection (no VPN/tunnel). It extends Runtipi's default setup with:
- Traefik v3 with custom middleware chains (rate limiting, security headers, Authentik ForwardAuth)
- Wildcard domain routing via Cloudflare DNS/ACME
- Per-app docker-compose overrides and environment variables

See `README.md` and `CREDITS.md` for provenance and attribution.

## Key Commands

```bash
# Start/stop Runtipi
sudo ./runtipi-cli start
sudo ./runtipi-cli stop

# Start a single app manually (without tipi CLI), example for an app override:
docker compose \
  --env-file app-data/jellyfin/app.env \
  --env-file user-config/jellyfin/app.env \
  --project-name jellyfin \
  -f apps/jellyfin/docker-compose.yml \
  -f repos/<hash>/apps/docker-compose.common.yml \
  -f user-config/jellyfin/docker-compose.yml \
  up --detach --force-recreate --remove-orphans

# Rename all .example files for first-time setup
find ./user-config -type f -name "*.example" | while read file; do
    mv "$file" "${file%.example}"
done
```

## Repository Structure

```
user-config/
├── tipi-compose.yml          # Top-level Docker Compose overrides (Traefik image, volumes, networks)
├── tipi.env                  # Global env vars for Traefik (secrets — not in git)
├── tipi.env.example          # Template for tipi.env
├── traefik/etc/
│   ├── traefik.yml           # Traefik static config (entrypoints, ACME, plugins)
│   └── dynamic/dynamic.yml   # Middleware definitions and chains (referenced via file provider)
└── rodrigomescua/<app>/
    ├── docker-compose.yml    # Service label overrides (Traefik routing/middleware)
    ├── app.env               # App-specific env vars (secrets — not in git)
    ├── app.env.example       # Template for app.env
    └── README.md             # App-specific Authentik setup instructions
```

Secrets (`tipi.env`, `app.env`, TLS keys) are git-ignored. Only `.example` counterparts are committed.

## Traefik Middleware Chains

Defined in `traefik/etc/dynamic/dynamic.yml` as Traefik file provider middleware chains:

| Chain | Usage |
|---|---|
| `chain-Domain-auth@file` | Internet-facing apps — rate limit + security headers + Authentik ForwardAuth |
| `chain-Domain@file` | Internet-facing apps with own auth (e.g. Nextcloud) — no ForwardAuth |
| `chain-localDomain-auth@file` | Local network + Authentik ForwardAuth |
| `chain-localDomain@file` | Local network only, no auth |

## Per-App Override Pattern

Each app under `rodrigomescua/<app>/docker-compose.yml` overrides Traefik labels generated by Runtipi. The minimal pattern for an internet-exposed app with Authentik ForwardAuth:

```yaml
services:
  <app-id>:
    labels:
      traefik.http.routers.<app-id>-rodrigomescua.middlewares: chain-Domain-auth@file
      traefik.http.routers.<app-id>-rodrigomescua.tls.certresolver: ''
```

For apps needing both internet access (with auth) and unauthenticated local network access, add a `-privip` router variant:

```yaml
      traefik.http.routers.<app-id>-rodrigomescua-privip.rule: Host(`${APP_DOMAIN}`) && ClientIP(`${PRIVATE_IPV4}`)
      traefik.http.routers.<app-id>-rodrigomescua-privip.middlewares: chain-Domain@file
```

The router name format is always `<app-id>-<username>` (e.g. `jellyfin-rodrigomescua`), matching what Runtipi generates.

## Environment Variables

Key variables in `tipi.env` / `tipi.env.example`:

- `WILDCARD_DOMAIN` — base domain (e.g. `heiland.cloud`)
- `CATCHALL_DOMAIN` — same value, used in per-app `app.env` for Authentik catchall routing
- `PRIVATE_IPV4` — local subnet CIDR for IP allowlist rules
- `CF_DNS_API_TOKEN` — Cloudflare DNS API token for ACME
- `BASICAUTH_DEFAULT_USER01` — htpasswd hash for Traefik dashboard basic auth

## Adding a New App

1. Create `rodrigomescua/<app>/docker-compose.yml` with Traefik label overrides
2. If the app needs extra env vars, create `rodrigomescua/<app>/app.env` (and commit an `app.env.example`)
3. Use `chain-Domain-auth@file` for ForwardAuth, `chain-Domain@file` for apps with their own auth
4. Restart the app in Runtipi to apply changes (regenerates `docker-compose.generated.yml`)
